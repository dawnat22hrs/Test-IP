# Withdraw Page - Frontend

## Запуск

Лучше использовать node 22й версии

1. `yarn install`
2. `yarn dev` - запускает приложение на `http://localhost:3000`
3. `yarn mock-server` - запускает моковый сервер на `http://localhost:4000`
4. `yarn test` - юнит тесты
5. `yarn cypress:open` - E2E тесты

## Retry и Idempotency

- Для сетевых ошибок (кроме 409) используется retry до 2 раз с задержкой 200ms.
- `POST /v1/withdrawals` отправляется с уникальным `Idempotency-Key`.
- Если приходит 409 — пользователь видит понятное сообщение "Заявка уже существует".
- Введенные данные сохраняются при повторной попытке.

## Безопасность токенов

- Access token хранится только в Redux (не в localStorage).
- Для продакшена: хранить в httpOnly cookie и использовать серверную авторизацию.
- Для development/моков можно генерировать временный UUID.

## Особенности

- Последняя заявка сохраняется и восстанавливается до 5 минут после reload.
- Мемоизация и `React.memo` для оптимизации рендеров.
- Styled-components + Theme для цветов и стилей.
